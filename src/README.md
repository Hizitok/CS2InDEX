# Project Overview

该项目实现了一个基于智能合约的去中心化交易系统，主要通过 Vault、Core、Engine 和 Router 四个合约进行资金管理、订单簿处理、清算操作以及用户交互。系统通过使用 NFT 来管理用户的仓位，以增强系统的透明度和便捷性。

---

## Contract Structure

- **Vault.sol**  
  资金托管合约。该合约负责管理用户的保证金和资产。用户的资金会通过该合约进行托管，确保资金的安全性和合约之间的正确交互。

- **Core.sol**  
  订单簿（OrderBook）核心合约。该合约负责存储和管理所有的买卖订单。订单簿使用平衡树数据结构来优化查询和订单匹配的效率。

- **Engine.sol**  
  资金费率与清算合约。该合约用于计算交易的资金费用和执行仓位清算。清算功能包括自动清算机制（ADL），用于维护账户的健康状态，避免因价格波动导致的过度风险。

- **Router.sol**  
  用户交互合约。用户通过该合约进行下单、平仓、查询仓位状态等操作。Router合约实现了与用户的交互逻辑，确保用户的操作能够被正确地发送到核心合约进行处理。

- **PositionNFT.sol**  
  仓位信息管理 核心合约。该合约负责存储和管理所有的仓位数据，在ERC721标准下仅可由Core合约进行创建和信息更新。

---

## 使用NFT管理仓位

该系统使用NFT作为每个用户仓位的代币表示。每个仓位对应一个唯一的NFT，确保仓位管理的透明性和去中心化。具体操作流程如下：

### 1. **开仓**

- 用户提交开仓请求，系统收取相应的保证金。
- 系统会铸造一个新的NFT，作为该仓位的唯一标识。
- 此NFT会被存储在用户账户中，且在**OrderBook**合约中挂单，等待匹配。
- 一旦订单成交，NFT的状态会被更新，标记为已成交，并进行相应的仓位调整。

### 2. **平仓**

- 用户提交平仓请求，系统会更新对应的NFT状态。
- 平仓完成后，NFT会显示为已平仓状态，并从订单簿中移除。
- 用户的资金会根据平仓结果返还至Vault合约。

## 数据结构

- **OrderBook平衡树** 存储于 **Core合约**  
  该平衡树数据结构用于高效存储和管理订单数据。它能快速处理订单的插入、查询和匹配，使交易过程更加高效。

- **ADL平衡树** 存储于 **Engine合约**  
  自动清算（ADL）平衡树用于存储和处理需要清算的仓位信息。ADL机制确保当账户的资金不足时，系统能够自动触发清算，防止账户爆仓。

---

## 操作流程

1. **开仓操作**  
   用户提交订单，系统验证保证金是否足够。如果满足条件，会铸造一个NFT，并在订单簿上挂单。订单的状态在成交后会更新。

2. **平仓操作**  
   用户提交平仓请求，系统会根据当前市场价格计算平仓金额，并更新NFT的状态。成功平仓后，用户的资金会返还至Vault合约，NFT会移除或标记为已平仓。

3. **自动清算**  
   在市场价格剧烈波动时，ADL机制会根据账户的资金状态触发清算，以防止账户出现负债。

---

## 合约间交互

- **Vault** 合约通过**Router**合约与用户交互，确保用户的资金在进行交易前被安全托管。
- **Core** 合约的订单簿与**Engine**合约的清算机制配合工作，确保订单的匹配和资金的安全。
- 用户通过**Router**合约提交的交易请求会首先进入**Core**合约的订单簿，等待匹配，匹配后订单状态更新，进而触发**Engine**合约的资金费率与清算操作。

---

## 安全性

- **资金托管**：所有资金都会存储在**Vault.sol**合约中，合约逻辑确保资金的安全，防止用户资金被篡改或挪用。
- **清算机制**：通过**Engine.sol**合约的自动清算（ADL）机制，系统能够在市场剧烈波动时自动执行仓位清算，防止账户过度亏损。
- **透明度**：使用NFT来表示仓位，每个NFT代表一个唯一的仓位，确保交易和资金流向的透明性。
