## Slide 2 — The Problem

### CS2 皮肤市场是一个 $50 亿美元的"被锁住的市场"

- **全球超 3500 万 CS2 活跃玩家**，许多人持有价值数千美元的皮肤资产
- 皮肤价格波动剧烈（±30-50%/年），但**持有者无法对冲**
- 传统做空机制缺失：借出皮肤周转受限、无衍生品市场

**Steam T+7 交易锁定机制——最具代表性的风险暴露场景：**

> Valve 规定，每笔 CS2 皮肤交易在steam市场以外成交后，买方在 **7 天内无法再次交易该物品**，且买卖双方在此期间均有权撤销交易,仅卖方撤回的交易回退资金给买方。这意味着：

```
场景：你以 $500 买入一把蝴蝶刀，7 天锁定期
  Day 0  ── 交易成交，$500 进入 Steam 余额
  Day 1  ── CS2 大更新，刀价暴涨 30%
  Day 3  ── 买家行使撤销权，刀被退回,你拿回$500
结果：你在最低点买入，因为对手方撤回交易错失了 $150 的涨幅，且毫无对冲手段
```

```
场景：你以 $500 买入一把刀，7 天锁定期
  Day 0  ── 买入完成，物品锁定
  Day 4  ── 市场下跌 20%，刀价变为 $400
  Day 7  ── 锁定期结束，账面亏损 $100
结果：持有 7 天强制暴露于价格风险，无任何对冲工具
```

对于买家而言，买入一个饰品，无论涨跌都有风险，理想的对冲工具有复合蝶式期权等。

**核心痛点：**
| 用户类型 | 具体场景 | 现有解决方案 |
|---------|---------|------------|
| 皮肤卖家 | T+7 锁定期内价格反弹，撤回交易导致买家亏损 | ❌ 无 |
| 皮肤买家 | T+7 锁定期内价格下跌，被迫亏损 | ❌ 无 |
| 大额持仓者 | 职业选手持有数万美元皮肤，熊市无法套保 | ❌ 无 |
| 游戏投机者 | 看跌 CS2 皮肤市场，无法做空 | ❌ 无 |
| 加密交易员 | 想接触游戏资产类别，无合规渠道 | ❌ 无 |

## Slide 2.5 — The Hedge Use Case

### 用统一指数对冲现货价格波动

**核心逻辑：持有现货皮肤 → 做空对应指数 → 锁定价值**

CS2 皮肤现货流动性极差（T+7 锁定、买家随时可撤单），持仓者几乎无法在价格下跌时及时止损。CS2InDEX 提供一个**随时可用、链上结算**的对冲渠道：

```
持有现货（刀具 $5,000）
        ↓
做空 CS2 Knives Index 永续合约（$5,000 空头，1x）
        ↓
  市场下跌 20%
        ↓
  现货亏损 $1,000  ←——→  空头盈利 $1,000
        ↓
净敞口归零，等待市场回稳后平仓
```

**为什么指数对冲有效？**

| 场景 | 无对冲 | 有对冲（做空指数） |
|------|--------|-----------------|
| 市场整体下跌 20% | 亏损 $1,000 | 净亏损 ≈ $0 |
| T+7 锁定期内暴跌 | 只能看着 | 永续合约 24/7 可交易 |
| 买家撤单，刀被退回 | 错过上涨，无法补救 | 平仓多单，锁定收益 |

> 单个皮肤价格与对应分类指数（刀具、步枪、手套）高度相关（β ≈ 0.8–0.95），使用指数作为对冲工具可捕获大部分系统性风险，剩余的个体风险（特定皮肤的稀有度溢价）远小于整体市场波动。

**⚠ 数据验证需求（Pre-Launch Research）：**

上述相关性假设需要通过历史数据实证：
(Pricempire有全item历史价格数据,需要买数据验证)

| 研究项 | 方法 | 数据来源 |
|--------|------|---------|
| 个体皮肤 vs. 分类指数的 β 系数 | 对 Buff163 历史价格做 OLS 回归，按 SKU 分组 | Buff163.com API（日线/周线） |
| 相关系数稳定性（熊/牛市分区间） | 滚动 30 天窗口计算 Pearson r，检验跨周期一致性 | PriceEmpire / Buff163 历史导出 |
| 对冲有效性回测 | 模拟持有现货 + 做空等额指数，计算净持仓波动率降幅 | 组合历史价格序列 |
| 极端行情下的相关性失效（Basis Risk） | 识别相关系数 < 0.5 的异常区间及触发条件 | 大版本更新 / CS2 重大事件节点 |

**对冲用户画像：**
- **职业皮肤商** —— 库存价值 $10K+，需要持续套保
- **大额持仓玩家** —— 职业选手 / 收藏家，持仓数万美元，熊市不愿低价抛售
- **交易所/平台方** —— 第三方皮肤交易平台对未成交订单做账面对冲

```

│                    用户（钱包）                      │

└────────────────────┬────────────────────────────────┘
                     │

│   Pool.sol — 链上订单簿（Order Statistics Tree）      │
│   • Limit / Market 订单                              │

│   • Maker 0.3% / Taker 0.5% 手续费                   │
└──────┬────────────────────────────┬─────────────────+
       │                            │
┌──────▼──────┐           ┌─────────▼────────┐
│PositionNFT  │           │ IndexOracle.sol   │
│ ERC721 头寸  │           │ VTWAP 价格聚合    │
│ 可转让所有权 │           │ 每 8h 资金费率结算│
└──────┬──────┘           └─────────┬────────┘
       │                            │
┌──────▼────────────────────────────▼────────┐
│            Liquidation.sol                 │
│   自动监控保证金率，触发平仓（20% 维持保证金）│
└────────────────────────────────────────────┘
                     ↕
┌────────────────────────────────────────────┐
│  Oracle Service（链下）                     │
│  SkinFlow.gg + EsportFire + Buff163        │
│  每 5 分钟聚合价格，推送到链上              │
└────────────────────────────────────────────┘
```

**技术亮点：**
- On-chain Order Statistics Tree（红黑树）：O(log n) 插入/删除，无 AMM 滑点
- VTWAP Oracle：min(Δt, 1h) × size 权重，抗价格操纵
- NFT 头寸：EIP-721 标准，头寸可在二级市场交易

## Appendix — FAQ

**Q: 为什么不直接做 CS2 皮肤现货 DEX？**
A: 皮肤是链下资产（Steam 生态），链上无法持有实物。指数衍生品绕过这一限制，同时提供杠杆和双向交易能力。

**Q: Oracle 被操纵怎么办？**
A: 双重保护：（1）链下聚合多个数据源取中位数；（2）链上 VTWAP 使用 min(Δt, 1h) × size 权重，单笔大单权重被时间稀释。

**Q: CS2 游戏关闭或皮肤系统变化怎么办？**
A: 同样适用于所有以游戏为底层资产的产品（如 Axie）。协议设计了紧急暂停机制。长期风险通过多游戏资产分散（Q3 计划）对冲。

**Q: 你们的优势能被 GMX 复制吗？**
A: 技术上可以，但 GMX 的核心优势（深度流动性、品牌效应）在加密货币市场。CS2 玩家不是 GMX 的用户。社区冷启动从游戏玩家侧进入更快。

**Q: 为什么在 Unichain 而不是以太坊主网？**
A: 订单簿每笔操作 ~200-300K gas，以太坊主网每笔 $3-10，用户不可接受。Unichain 降低 10-50x，使高频交易成为可能。
